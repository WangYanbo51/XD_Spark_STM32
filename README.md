# 基于姿态识别的无线手势控制智能小车系统
![Python](https://img.shields.io/badge/Python-3.7+-blue.svg) ![C](https://img.shields.io/badge/C-STM32F103%2F401-green.svg) ![IDE](https://img.shields.io/badge/IDE-STM32CubeIDE-orange.svg) ![License](https://img.shields.io/badge/License-GPL3.0-green.svg)

低成本手势控制智能小车 | 手套+小车双STM32主控 | 上位机3D姿态可视化 | 星火杯参赛作品

## 📖 项目概述
通过手套端采集手部姿态，无线传输至小车端，实现手势直接控制小车运动；配套Python上位机，实时3D可视化姿态，适用于嵌入式DIY、实训场景。

核心配置：小车（STM32F103C8T6）+ 手套（STM32F401RCT6），STM32CubeIDE开发，MPU6050姿态采集，HC-12无线传输。

## 🔧 硬件配置
| 模块 | 规格详情 |
|------|----------|
| 小车主控 | STM32F103C8T6 |
| 手套主控 | STM32F401RCT6 |
| 姿态采集 | MPU6050六轴传感器（手套端） |
| 无线传输 | HC-12无线串口模块（波特率115200） |
| 电机驱动 | HW-856双驱动板 + 直流电机（小车端） |
| 上位机 | PC端（Python 3.7+） |

## ✨ 核心功能
- MPU6050采集手部姿态，卡尔曼滤波降噪，精准识别前进/转向/刹车等指令
- HC-12无线双向通信，延迟≤10ms，空旷传输距离≤1000米
- 小车四轮同步驱动，0~999级线性调速，动作无卡顿
- Python上位机3D可视化，同步还原手部姿态，支持抗锯齿、高分辨率

## 💻 快速开始
### 环境准备
- Python 3.7+：安装依赖 pip install pyserial pyopengl
- 开发工具：STM32CubeIDE（适配STM32F103C8T6、STM32F401RCT6）
- 硬件接线：按上述硬件配置连接，确保HC-12波特率为115200

### 部署步骤
1. 克隆仓库：git clone 仓库地址
2. 硬件接线：对照项目引脚图，分别完成小车端（STM32F103C8T6）与电机驱动板、HC-12模块的接线，以及手套端（STM32F401RCT6）与MPU6050、HC-12模块的接线
3. 程序编译与下载：在STM32CubeIDE中分别打开小车端（STM32F103C8T6）、手套端（STM32F401RCT6）项目，编译程序后，分别下载至对应单片机
4. 调试（可选）：若需调试手套端姿态采集精度，可运行上位机程序（python STM_Glove.py），实时观测STM32F401RCT6的姿态数据，优化滤波参数
5. 启动控制：调试完成后，给小车端、手套端供电，佩戴手套即可通过手部姿态控制小车运动

## 📊 性能参数
- 姿态解算：三轴误差≤1°，滤波后抖动≤0.5°
- 无线传输：延迟≤10ms，空旷距离≤1000米
- 小车控制：0~999级线性调速，转向/自转流畅
- 可视化：3D渲染帧率60FPS，支持720P/1080P、抗锯齿
